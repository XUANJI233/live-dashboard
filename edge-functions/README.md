# ESA 边缘计算 — Live Dashboard

## 这是什么？

把一部分计算任务从服务器搬到离用户最近的「边缘节点」，让页面加载更快、服务器压力更小。

## 解决了什么问题？

| 问题 | 原因 | 边缘计算如何解决 |
|------|------|----------------|
| 访客看到 403 错误 | CDN 缓存了 PoW 挑战 | PoW 在边缘生成，不再被缓存 |
| 服务器压力大 | 每次请求都回源 | 读取数据在边缘缓存，80% 请求不用回源 |
| 身份验证慢 | 每次都要问服务器 | Token 在边缘验证，无需回源 |

## 效果

- PoW 403 错误：**100% → 0%**
- 服务器请求量：**降低 80-90%**
- 页面响应：**更快**（数据从最近的边缘节点返回）

## 部署步骤

### 第 1 步：上传边缘函数

1. 登录 [阿里云 ESA 控制台](https://esa.console.aliyun.com/)
2. 进入 **边缘函数** → **创建函数**
3. 函数名称填 `live-dashboard-edge`
4. 把 `edge-router.js` 的代码粘贴进去

### 第 2 步：配置环境变量

在 ESA 函数设置里添加：

| 变量 | 填什么 |
|------|--------|
| `ORIGIN_URL` | `http://172.20.0.80:3000`（你的服务器内网地址） |
| `HASH_SECRET` | 和服务器一样的密钥（用于签名验证） |
| `EDGE_KV_NAMESPACE` | `live-dashboard` |

### 第 3 步：创建存储空间

1. ESA 控制台 → **边缘存储** → **创建命名空间**
2. 名称填 `live-dashboard`

### 第 4 步：配置路由

ESA 控制台 → **函数路由** 添加：

- `live.myallinone.online/api/*` → 函数 `live-dashboard-edge`

### 第 5 步：源站（无需操作）

源站会自动识别边缘函数的请求，不需要改任何配置。只需确保 `HASH_SECRET` 和边缘函数一致。

## 安全说明

- 密钥 `HASH_SECRET` 不在代码里，通过环境变量配置，不会泄露
- 只有知道密钥的人才能部署边缘函数

## 限流

边缘函数比服务器限流更宽松（因为边缘分担了压力）：

- 每个 IP 每分钟最多 300 次请求
- 访客验证每分钟最多 12 次
- 已登录用户每分钟最多 60 次

## 注意事项

- 边缘存储是全球分布的，数据同步可能有几秒延迟
- 边缘函数单次执行最多 10 秒
